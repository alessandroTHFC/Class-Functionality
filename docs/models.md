# Eloquent Models

All models live in `backend/app/Models/`. The project uses **single-database multi-tenancy**: all tenants share one MySQL database and every tenant-scoped table carries a `tenant_id` column. Stancl's `BelongsToTenant` trait adds a global Eloquent scope that automatically appends `WHERE tenant_id = ?` to every query once the tenant context is initialised.

---

## Tenant-Scoped Models (all carry tenant_id)

All models below use the `BelongsToTenant` trait. Stancl automatically sets `tenant_id` on create and filters by it on all reads. The tenant context is initialised from the authenticated user's `tenant_id` via a custom middleware.

### User
**Table:** `users`

> `email` is **globally unique** across all tenants. This is what makes tenant-from-user auth possible â€” a login lookup by email always resolves to exactly one user and one tenant.

| Field | Type | Nullable | Notes |
|---|---|---|---|
| id | bigint (auto) | No | Primary key |
| tenant_id | string | No | FK â†’ tenants.id |
| name | string | No | Full display name |
| email | string | No | Globally unique (unique index, no tenant scope) |
| email_verified_at | timestamp | Yes | |
| password | string | No | Bcrypt hashed |
| remember_token | string | Yes | |
| created_at | timestamp | Yes | |
| updated_at | timestamp | Yes | |
| deleted_at | timestamp | Yes | SoftDeletes |

**Relationships**
- `tenant()` â†’ belongsTo `Tenant`
- `assignedClasses()` â†’ belongsToMany `SchoolClass` via `class_users`
- `createdClasses()` â†’ hasMany `SchoolClass` (FK: `created_by_user_id`)
- `notes()` â†’ hasMany `StudentNote`
- `roles()` â†’ belongsToMany (Spatie)

**Traits:** `SoftDeletes`, `HasRoles` (Spatie), `BelongsToTenant` (Stancl)
