# Eloquent Models

All models live in `backend/app/Models/`. The project uses **single-database multi-tenancy**: all tenants share one MySQL database and every tenant-scoped table carries a `tenant_id` column. Stancl's `BelongsToTenant` trait adds a global Eloquent scope that automatically appends `WHERE tenant_id = ?` to every query once the tenant context is initialised.

---

## Central Models (no tenant_id)

These tables exist in the shared database and are never tenant-scoped.

### Tenant
**Table:** `tenants`
Managed by Stancl. Represents one school.

| Field | Type | Nullable | Notes |
|---|---|---|---|
| id | string (ulid) | No | Primary key, generated by Stancl |
| name | string | No | School display name |
| data | json | Yes | Stancl metadata |
| created_at | timestamp | Yes | |
| updated_at | timestamp | Yes | |

**Relationships**
- `domains()` → hasMany `Domain`

## Tenant-Scoped Models (all carry tenant_id)

All models below use the `BelongsToTenant` trait. Stancl automatically sets `tenant_id` on create and filters by it on all reads. The tenant context is initialised from the authenticated user's `tenant_id` via a custom middleware.

### User
**Table:** `users`

> `email` is **globally unique** across all tenants. This is what makes tenant-from-user auth possible — a login lookup by email always resolves to exactly one user and one tenant.

| Field | Type | Nullable | Notes |
|---|---|---|---|
| id | bigint (auto) | No | Primary key |
| tenant_id | string | No | FK → tenants.id |
| name | string | No | Full display name |
| email | string | No | Globally unique (unique index, no tenant scope) |
| email_verified_at | timestamp | Yes | |
| password | string | No | Bcrypt hashed |
| remember_token | string | Yes | |
| created_at | timestamp | Yes | |
| updated_at | timestamp | Yes | |
| deleted_at | timestamp | Yes | SoftDeletes |

**Relationships**
- `tenant()` → belongsTo `Tenant`
- `assignedClasses()` → belongsToMany `SchoolClass` via `class_users`
- `createdClasses()` → hasMany `SchoolClass` (FK: `created_by_user_id`)
- `notes()` → hasMany `StudentNote`
- `roles()` → belongsToMany (Spatie)

**Traits:** `SoftDeletes`, `HasRoles` (Spatie), `BelongsToTenant` (Stancl)

### SchoolClass
**Table:** `classes`

> Named `SchoolClass` to avoid conflict with PHP's reserved `class` keyword.

| Field | Type | Nullable | Notes |
|---|---|---|---|
| id | bigint (auto) | No | Primary key |
| tenant_id | string | No | FK → tenants.id |
| name | string | No | e.g. "Year 9 Science" |
| year_level_id | bigint | Yes | FK → year_levels.id |
| created_by_user_id | bigint | No | FK → users.id |
| created_at | timestamp | Yes | |
| updated_at | timestamp | Yes | |
| deleted_at | timestamp | Yes | SoftDeletes |

**Relationships**
- `yearLevel()` → belongsTo `YearLevel`
- `createdBy()` → belongsTo `User` (FK: `created_by_user_id`)
- `users()` → belongsToMany `User` via `class_users`
- `students()` → belongsToMany `Student` via `class_students`
- `notes()` → hasMany `StudentNote`

**Traits:** `SoftDeletes`, `BelongsToTenant`

**Scopes**
- `scopeAssignedTo($query, User $user)` — filters to classes where the user appears in `class_users`
- `scopeSearch($query, string $term)` — filters by name using LIKE

### ClassUser
**Table:** `class_users`
Pivot between `SchoolClass` and `User`. Defined as a full Eloquent model to allow direct querying.

| Field | Type | Nullable | Notes |
|---|---|---|---|
| id | bigint (auto) | No | Primary key |
| class_id | bigint | No | FK → classes.id |
| user_id | bigint | No | FK → users.id |
| created_at | timestamp | Yes | |
| updated_at | timestamp | Yes | |

> No `tenant_id` on this table. Tenant isolation is inherited through `class_id` — every class already belongs to one tenant and is query-scoped by `BelongsToTenant` on `SchoolClass`. Direct queries on `ClassUser` without going through a class are not used in this application.

**Traits:** none

---

### Student
**Table:** `students`

| Field | Type | Nullable | Notes |
|---|---|---|---|
| id | bigint (auto) | No | Primary key |
| tenant_id | string | No | FK → tenants.id |
| given_name | string | No | |
| family_name | string | No | |
| date_of_birth | date | Yes | |
| year_level_id | bigint | Yes | FK → year_levels.id |
| nccd_level | string | Yes | Enum: QDTP, Supplementary, Substantial, Extensive |
| nccd_category | string | Yes | Enum: Cognitive, Physical, Sensory, Social/Emotional |
| primary_disability | string | Yes | Free text |
| primary_disability_level_formalised | boolean | No | Default: false |
| created_at | timestamp | Yes | |
| updated_at | timestamp | Yes | |
| deleted_at | timestamp | Yes | SoftDeletes |

**Relationships**
- `yearLevel()` → belongsTo `YearLevel`
- `classes()` → belongsToMany `SchoolClass` via `class_students`
- `notes()` → hasMany `StudentNote`

**Traits:** `SoftDeletes`, `BelongsToTenant`

**Casts**
```php
protected $casts = [
    'date_of_birth'                       => 'date',
    'primary_disability_level_formalised' => 'boolean',
    'nccd_level'                          => NccdLevelEnum::class,
    'nccd_category'                       => NccdCategoryEnum::class,
];
```

**Appended Accessors**
- `full_name` → `"{$this->given_name} {$this->family_name}"`
